Intro

The company was seeking to manage their claims for untracked transactions better. A lot of customers were still contacting customer support asking for help with that, which played a big role in increased costs and of course needed the UX improved. The old approach incorporated a 5 step form, and looking at data, it was clear that a significant amount of users were dropping off after step 3.

We use firebase in this team, so I was looking through the data to draw up some conclusions and findings.

Goal

To make the claims process quicker, and easier to understand, while still keeping to strict development deadlines.

Research

We released a survey for users to leave feedback after they have used the claims service (even if they dropped off before finishing it). The results were overall quite positive from the users that took part, however, we are considering all of the data.

I started by looking at current flows to find flaws and good points in the current structure. I also had to liaise with the developers to see which solutions we can potentially implement.

Looking at user feedback, user data, development requirements, paint points and positive points, and post workshops held with the team, I drew up a new proposed flow that potentially simplifies the process. The key here was to not reinvent the whole process, but to simplify the current flow so we could test and roll this out to meet deadlines.

The new proposed flows were wireframed and presented to the Product manager. There were some fundamental issues such as - user gets taken to a web page for some more information, but they need to log in again, which may have been one of the contributors to users dropping off towards the end of the claims process.

Looking at the complete flow from start to finish (to email etc)

User

I drew up the user cohorts, just so we can understand and test how a new user might approach this journey against an existing user and making sure the flow fits all instances and how the claims process might be useful to each of them.

And using the current findings I was able to wireframe the updated flows and the multiple entry points all users may come across.

Constraints and scope

We had a very limited deadline for this project, so my main focus was on what we can do in the time that we have. There were also dev limitations so I had to account for those in my research and designs. The end result was an improved and condensed version of the existing process.
